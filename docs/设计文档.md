# 股票 K 线复盘训练系统设计文档

## 1. 项目定位

这是一个面向个人投资训练的股票/ETF K 线复盘网站。它不负责荐股，也不替代真实交易软件，而是用历史行情模拟“当时只能看到当时信息”的买卖过程，训练用户在价格波动、浮亏、指标变化和交易笔记之间建立真实感。

核心训练原则：

- 只使用今天以前的历史 K 线数据，避免把未来走势带进决策。
- 复盘时隐藏未来 K 线，用户只能逐日推进。
- 买入按所选交易日最高价成交，卖出按所选交易日最低价成交，用保守价格训练择时。
- 买入后持续显示到当前复盘日的浮动亏损、最大浮亏和总盈亏，让模拟过程也有压力反馈。
- 每个买卖点都能记录理由、情绪、指标判断和事后复盘。

## 2. 现有 demo 评估

当前目录是一个纯静态 demo：

- `index.html`：页面骨架。
- `app.js`：内置假数据、指标计算、Canvas 绘图、交易记录和 localStorage。
- `styles.css`：基础布局和视觉样式。
- `serve.mjs`：本地静态服务。

已经验证的体验：

- 股票/ETF 搜索和加入自选。
- K 线、成交量、MA、BOLL、KDJ、MACD 展示。
- 自定义 MA 周期。
- 隐藏未来和逐日复盘。
- 买入按最高价、卖出按最低价。
- 交易笔记、持仓、平均成本、已实现盈亏、浮动盈亏、最大浮亏和总盈亏。

主要缺口：

- 使用假数据，不具备真实股票/ETF历史行情。
- 所有逻辑集中在一个 JS 文件，后续维护成本高。
- Canvas 手绘图表后续扩展十字光标、多窗格、缩放、指标面板、标记点会变复杂。
- localStorage 只能用于个人原型，不能承载长期数据、同步和统计。
- 当前中文在终端输出里有乱码现象，正式开发前应统一文件编码为 UTF-8。

结论：现有 demo 可以作为需求原型和交互参考，但建议进入正式开发时保留体验，重建技术结构。

## 3. 推荐技术选型

### 3.1 总体建议

建议采用“前端 React + TypeScript + KLineCharts，后端 FastAPI + SQLite/SQLModel，行情数据先接 AKShare，预留 Tushare Pro”的路线。

这样做的原因：

- 你要做的是个人训练工具，不是高并发金融终端，第一阶段应该快速成型。
- A 股和 ETF 数据接入在 Python 生态更方便，AKShare/Tushare 都主要面向 Python。
- K 线图、指标、多窗格、覆盖线和买卖标记是项目核心，应该优先使用专业 K 线库，减少手写图表成本。
- SQLite 足够支撑单机和个人使用，未来需要多设备同步时再迁移 PostgreSQL。

### 3.2 技术栈表

| 模块 | 推荐 | 说明 |
| --- | --- | --- |
| 前端框架 | React + TypeScript + Vite | 开发快，类型明确，适合把当前 demo 拆成组件 |
| K 线图表 | KLineCharts | 内置常见技术指标，适合 MA、VOL、MACD、KDJ、BOLL 和覆盖标记 |
| UI 状态 | Zustand | 轻量管理自选、复盘日期、指标开关、选中交易 |
| 表单校验 | React Hook Form + Zod | 用于交易表单、笔记、参数配置 |
| 后端 API | FastAPI | Python 生态友好，适合接行情源、做缓存、计算指标 |
| ORM/模型 | SQLModel | 同时服务数据库模型和接口类型，适合 FastAPI |
| 本地数据库 | SQLite | 第一阶段简单可靠；后续可迁移 PostgreSQL |
| 行情数据源 | AKShare 优先，Tushare Pro 作为稳定备选 | AKShare 适合快速开发；Tushare Pro 需要 token/积分但更适合长期稳定 |
| 定时任务 | APScheduler | 每个交易日收盘后增量同步历史行情 |
| 测试 | Vitest + Playwright + pytest | 前端单测、端到端交互、后端计算规则测试 |

### 3.3 图表库对比

| 方案 | 优点 | 缺点 | 结论 |
| --- | --- | --- | --- |
| KLineCharts | 面向 K 线，内置指标和覆盖能力，上手最贴近需求 | 生态比 ECharts 小 | 推荐主选 |
| Apache ECharts | 通用图表强、文档多、可高度定制 | 金融复盘交互要自己搭较多逻辑 | 可作为备选 |
| TradingView Lightweight Charts | 性能好、体积小、金融图表体验好 | 指标需要自行计算和组织多窗格 | 适合后续更精细自定义时评估 |
| 当前 Canvas 手绘 | 无依赖，可完全控制 | 长期维护和交互扩展成本高 | 只保留为原型参考 |

## 4. 功能范围

### 4.1 第一阶段 MVP

第一阶段必须让用户完成一次完整复盘，并形成“复盘工作台 + 工具箱”的基础信息架构：

- 左侧图标侧边栏菜单。
- 点击不同菜单图标切换不同页面内容。
- 复盘工作台作为默认首页。
- 工具箱页面提供常用交易计算器。
- 搜索股票/ETF。
- 加入自选。
- 拉取并缓存历史日 K。
- 选择复盘开始日期。
- 隐藏未来 K 线并逐日推进。
- 显示 MA、BOLL、KDJ、MACD、成交量。
- 在指定日期买入或卖出。
- 买入成交价为当日最高价，卖出成交价为当日最低价。
- 记录交易笔记。
- 显示持仓、平均成本、已实现盈亏、浮动盈亏、最大浮亏和总盈亏。
- 在 K 线上显示买卖点标记。
- 利润成本计算器：按股票/ETF 计算买卖成本、净盈亏和净利润率。
- 涨跌幅计算器：按价格和涨跌幅互相换算。
- 平均价格计算器：按多笔买入计算平均成本。

### 4.2 第二阶段增强

- 区间复盘：选择某一段买入卖出过程，单独计算收益、亏损和回撤。
- 笔记标签：追高、恐慌、纪律、趋势、指标背离、仓位过重等。
- 复盘模式：逐日播放、暂停、快进、回退。
- 交易截图：保存某次买卖时的 K 线状态和指标状态。
- 错因统计：按标签统计亏损来源。
- 策略评分：是否按计划、是否提前卖出、是否追涨杀跌。
- 做 T 计算器：根据底仓、当日买入、当日卖出计算做 T 后成本、现金流和当日 T 盈亏。
- 计算器历史记录：保存常用计算参数和结果。

### 4.3 第三阶段可选

- 多账户/多复盘计划。
- 云同步和登录。
- 分钟线复盘。
- 真实交易导入。
- 训练题库：系统随机选择一段历史行情，让用户盲测操作。
- AI 复盘助手：根据用户笔记和交易记录总结常见问题。

## 4.4 工具箱计算器需求

工具箱是复盘系统的辅助区域，不直接替代复盘交易记录。第一阶段建议全部在前端计算，后续如果需要保存历史记录，再接入后端。

### 4.4.1 利润成本计算器

输入：

- 成本类型：股票 / ETF。
- 买入价格。
- 卖出价格。
- 买入数量。
- 佣金费率。
- 最低佣金。
- 印花税率，股票默认卖出收取，ETF 默认 0。
- 过户费率，可选。

输出表格：

| 字段 | 说明 |
| --- | --- |
| 买入金额 | 买入价格 * 买入数量 |
| 买入佣金 | max(买入金额 * 佣金费率, 最低佣金) |
| 买入合计 | 买入金额 + 买入佣金 + 买入过户费 |
| 卖出金额 | 卖出价格 * 卖出数量 |
| 卖出佣金 | max(卖出金额 * 佣金费率, 最低佣金) |
| 卖出印花税 | 股票卖出金额 * 印花税率；ETF 默认 0 |
| 卖出合计 | 卖出金额 - 卖出佣金 - 卖出印花税 - 卖出过户费 |
| 净盈亏金额 | 卖出合计 - 买入合计 |
| 净利润率 | 净盈亏金额 / 买入合计 |
| 交易成本合计 | 买入佣金 + 卖出佣金 + 卖出印花税 + 过户费 |

默认规则：

- 股票默认佣金费率 0.025%，最低 5 元，卖出印花税 0.05%，过户费可先设为 0。
- ETF 默认佣金费率 0.025%，最低 5 元，无印花税。
- 费率必须允许用户修改，因为不同券商规则不同。

### 4.4.2 做 T 计算器

输入：

- 原持仓数量。
- 原平均成本。
- 当日买入价格。
- 当日买入数量。
- 当日卖出价格。
- 当日卖出数量。
- 成本类型：股票 / ETF。
- 佣金、印花税和过户费配置。

输出：

- 当日买入金额和费用。
- 当日卖出金额和费用。
- 做 T 净现金流。
- 做 T 已实现盈亏。
- 做 T 后剩余持仓数量。
- 做 T 后新的持仓成本。
- 做 T 后新的平均成本。

规则：

- 如果先买后卖，卖出部分优先按当日买入数量计算 T 盈亏，超出部分再按原持仓成本计算。
- 如果先卖后买，先按原持仓成本计算已实现盈亏，再用买入调整新成本。
- 第一版可以先要求用户选择“先买后卖”或“先卖后买”。

### 4.4.3 涨跌幅计算器

输入：

- 基准价格。
- 当前价格，或目标涨跌幅。

输出：

- 涨跌金额。
- 涨跌幅。
- 上涨到目标涨幅所需价格。
- 下跌到目标跌幅所需价格。
- 可选显示 A 股常见涨跌停价格，例如 10%、20%、5%。

### 4.4.4 平均价格计算器

输入：

- 多笔买入价格。
- 每笔买入数量。
- 每笔手续费，可选。

输出：

- 总数量。
- 总买入金额。
- 总交易费用。
- 加权平均价格。
- 含费用平均成本。

交互要求：

- 可以新增/删除一行买入记录。
- 输入变化后可实时计算，也保留“计算”按钮用于明确触发。
- 表格结果可以复制。

## 5. 核心业务规则

### 5.1 数据时间规则

- 系统只拉取今天以前的数据。
- 如果今天不是交易日，最新 K 线为最近一个交易日。
- 复盘时，图表最多展示到当前复盘日期。
- 所有指标在某一天的值只能由该日及之前的数据计算，禁止未来函数。

### 5.2 交易成交规则

买入：

```text
buy_price = selected_bar.high
```

卖出：

```text
sell_price = selected_bar.low
```

规则说明：

- 买入用最高价，是为了模拟“买得很差”的保守训练。
- 卖出用最低价，是为了模拟“卖得很差”的保守训练。
- 数量默认按 A 股习惯使用 100 股整数倍，但 ETF 也可以允许 1 份或 100 份配置。
- 不允许卖出超过当前持仓，除非后续明确支持做空。
- 手续费、印花税、过户费先做可配置项，MVP 可先支持固定手续费。

### 5.3 盈亏计算规则

建议第一阶段使用 FIFO 批次法：

- 每次买入形成一个 lot。
- 卖出时从最早买入 lot 开始扣减。
- 已实现盈亏 = 卖出收入 - 被扣减买入成本 - 卖出费用。
- 当前持仓成本 = 未卖出 lot 的买入成本 + 分摊费用。
- 平均成本 = 当前持仓成本 / 当前持仓数量。

浮动盈亏建议同时展示两种：

```text
当前收盘浮盈亏 = (当前复盘日 close - 平均成本) * 持仓数量
当前低点浮盈亏 = (当前复盘日 low - 平均成本) * 持仓数量
```

训练压力主要使用“当前低点浮盈亏”和“买入后最低低点浮盈亏”：

```text
最大浮亏 = (买入后至当前复盘日最低 low - 平均成本) * 当前持仓数量
总盈亏 = 已实现盈亏 + 当前低点浮盈亏
```

### 5.4 区间盈利计算

用户可以选中一段交易过程：

- 起点：某个买入记录。
- 终点：某个卖出记录，或当前复盘日。
- 系统计算该区间投入本金、卖出收入、区间已实现盈亏、区间浮动盈亏、最大浮亏和收益率。
- 区间可以附加复盘笔记，用于总结这一次完整交易。

## 8. 前端页面设计

### 8.1 主页面布局

建议从当前 demo 的工作台结构升级为“窄图标导航 + 页面内容区”的结构：

- 最左侧：窄侧边栏图标菜单，用于切换主页面。
- 复盘页左侧：搜索、自选、指标设置、复盘列表。
- 复盘页中间：K 线图表、指标窗格、买卖标记、十字光标信息。
- 复盘页右侧或底部：交易表单、盈亏面板、浮亏压力条、交易记录、区间复盘笔记。
- 工具箱页：利润成本计算器、做 T 计算器、涨跌幅计算器、平均价格计算器。

### 8.1.1 图标侧边栏菜单

菜单建议保持窄而稳定，避免占用 K 线分析空间。

建议菜单项：

| 菜单 | 图标建议 | 页面 |
| --- | --- | --- |
| 复盘 | CandlestickChart | K 线复盘训练台 |
| 计算器 | Calculator | 利润成本、做 T、涨跌幅、平均价格 |
| 统计 | BarChart3 | 胜率、盈亏比、错因统计，后续阶段 |
| 笔记 | NotebookPen | 交易笔记和区间复盘，后续阶段 |
| 设置 | Settings | 数据源、手续费、复权、主题配置 |

交互要求：

- 点击图标切换主内容区。
- 当前菜单高亮。
- 图标必须有 tooltip 或辅助文本。
- 桌面端默认只显示图标，悬停显示说明。
- 移动端可切换为底部导航或抽屉菜单。
- 复盘页内部的搜索、自选、指标设置仍然保留在复盘页的二级侧栏。

### 8.1.2 工具箱页面

工具箱页面使用 tabs 或分段控制切换计算器：

- 利润成本。
- 做 T。
- 涨跌幅。
- 平均价格。

每个计算器建议采用左输入、右结果的布局。结果表格要可读、可复制，并明确显示所使用的费率规则。

### 8.2 图表交互

必须支持：

- 缩放和平移。
- 十字光标显示 OHLC、成交量、KDJ、MACD、BOLL、MA。
- 买点和卖点明显标记。
- 买入后低点浮亏区域或压力线。
- 当前复盘日竖线。
- 隐藏未来时，未来 K 线不可见。

### 8.3 指标设置

第一阶段：

- MA 周期：默认 5、10、20，可修改。
- BOLL：默认 20，标准差 2。
- KDJ：默认 9、3、3。
- MACD：默认 12、26、9。
- 成交量显示开关。

第二阶段：

- 保存指标模板。
- 每个复盘 session 独立保存指标配置。

## 10. 数据源策略

第一阶段建议：

- 开发环境优先使用 AKShare。
- 股票日 K 可使用 A 股历史行情接口。
- ETF 历史行情可使用 ETF 历史行情接口。
- 将拉到的数据缓存进 SQLite，前端只访问自己的后端。

第二阶段建议：

- 增加 Tushare Pro 适配层。
- 在配置中选择数据源：`MARKET_DATA_PROVIDER=akshare|tushare`。
- 对同一个标的保存数据源、复权类型和同步时间。

注意：

- 不建议前端直接调用东方财富等站点接口，容易遇到跨域、接口变化和版权风险。
- 行情源可能有缺失、复权差异、接口变更，必须有缓存、重试和数据校验。
- 复盘训练对“历史是否可复现”要求高，因此不要每次页面打开都直接请求外部数据，应先落库。

## 11. 指标公式约定

### 11.1 MA

```text
MA(n) = 最近 n 个收盘价的简单平均
```

### 11.2 BOLL

默认参数：

```text
MID = MA(20)
UPPER = MID + 2 * STD(close, 20)
LOWER = MID - 2 * STD(close, 20)
```

### 11.3 KDJ

默认参数 9、3、3：

```text
RSV = (close - lowest(low, 9)) / (highest(high, 9) - lowest(low, 9)) * 100
K = 2/3 * previous_K + 1/3 * RSV
D = 2/3 * previous_D + 1/3 * K
J = 3 * K - 2 * D
```

### 11.4 MACD

默认参数 12、26、9：

```text
DIF = EMA(close, 12) - EMA(close, 26)
DEA = EMA(DIF, 9)
MACD_BAR = 2 * (DIF - DEA)
```

## 12. 风险与约束

- 数据源授权：个人学习和本地使用问题较小，但公开部署时要确认数据授权。
- 复权口径：前复权、后复权、不复权会影响价格和盈亏结果，必须在界面显示。
- 未来函数：指标和图表必须严格按复盘日截断。
- 交易规则：买高卖低是训练规则，不是现实撮合模型，要在产品中明确。
- 情绪训练：亏损压力展示要清晰，但不要过度刺激；建议用“压力、回撤、纪律”语言，而不是恐吓式文案。
- 数据更新：每天收盘后的数据更新存在延迟，应显示最后同步时间。

## 13. 优化想法

- 增加“盲测模式”：隐藏股票名称，只给 K 线和指标，训练纯图形判断。
- 增加“纪律分”：买卖前必须写计划，卖出后系统比较计划和实际行为。
- 增加“浮亏日记”：买入后每天记录当日情绪，和价格波动对照。
- 增加“错误标签热力图”：统计亏损最多来自哪些行为。
- 增加“复盘卡片导出”：把一笔完整交易生成图片或 Markdown，便于长期复习。
- 增加“只看亏损样本”：专门训练面对连续浮亏时的仓位和止损判断。
- 增加“计算器结果带入复盘”：利润成本计算器或做 T 计算器的结果可以一键生成一条复盘备注。
- 增加“费率模板”：按不同券商保存佣金、最低佣金、印花税、过户费配置。
