# 开发内容顺序

本文档按“每个阶段都能交付、验证、继续迭代”的方式组织。当前正式数据库路线改为 PostgreSQL，SQLite 仅作为临时测试或 demo 备选，不再作为主线数据库。

## 阶段 0：整理当前 demo

目标：保护当前原型，明确它的价值和限制。

开发内容：

- 确认旧 demo 可以运行。
- 统一文件编码为 UTF-8，修复中文乱码。
- 将旧 demo 移入 `prototype/`，作为交互参考。
- 记录当前 demo 已具备的复盘、交易、笔记、盈亏能力。

验收标准：

- 原型可以独立打开或运行。
- 页面中文正常显示。
- 能完成一次假数据买入、卖出、笔记和盈亏查看。

## 阶段 1：建立正式项目骨架

目标：从静态 demo 进入可维护工程结构。

推荐结构：

```text
apps/
  web/
  api/
docs/
prototype/
```

开发内容：

- 建立 `apps/web`：Vite + React + TypeScript。
- 建立 `apps/api`：FastAPI。
- 建立 `docs/`：放入 `设计文档.md` 和 `开发内容顺序.md`。
- 配置前后端基础启动脚本。
- 保留旧 demo 到 `prototype/`。
- 建立基础 README，说明启动方式。

验收标准：

- 前端能启动并显示工作台。
- 后端能启动并返回 `GET /api/health`。
- README 能指导从零启动项目。

## 阶段 2：建立前端信息架构与图标侧边栏

目标：稳定页面骨架，让复盘、计算器、统计、笔记、设置都有入口。

开发内容：

- 接入 `lucide-react`。
- 建立最左侧窄图标菜单。
- 菜单包含：复盘、计算器、统计、笔记、设置。
- 点击图标切换主内容区。
- 当前菜单高亮。
- 每个图标有 tooltip 或辅助文本。
- 建立前端页面路径：
  - `/replay`
  - `/calculators`
  - `/stats`
  - `/notes`
  - `/settings`
- 做基础移动端适配。

验收标准：

- 左侧能看到稳定的图标导航栏。
- 点击不同图标能切换页面内容。
- 刷新后默认进入复盘页。
- 图标不会挤压主工作区。

## 阶段 3：迁移复盘工作台布局

目标：把 demo 的复盘页面形态迁移到正式 React 页面。

开发内容：

- 拆分复盘页组件：
  - `ReplayPage`
  - `InstrumentSearch`
  - `Watchlist`
  - `IndicatorSettings`
  - `ChartWorkspace`
  - `TradePanel`
  - `PnlPanel`
  - `TradeHistory`
- 暂时使用 mock 数据替代真实接口。
- 保留“隐藏未来”“前一天/后一天”“交易笔记”等核心控件。
- 用前端状态保存当前标的、复盘日期、指标配置和交易记录。

验收标准：

- 复盘页布局接近 demo。
- 自选、指标开关、复盘日期在刷新前可正常交互。
- 买入、卖出、笔记和盈亏面板可用。

## 阶段 4：实现工具箱计算器 MVP

目标：把常用交易计算工具做成可用页面。

开发内容：

- 建立 `CalculatorsPage`。
- 使用 tabs 或分段控件切换：
  - 利润成本计算器
  - 做 T 计算器
  - 涨跌幅计算器
  - 平均价格计算器
- 建立统一费率配置：
  - 成本类型：股票 / ETF
  - 佣金费率
  - 最低佣金
  - 印花税率
  - 过户费率
- 所有计算逻辑抽成纯函数。

验收标准：

- 股票默认卖出收印花税，ETF 默认不收印花税。
- 点击或修改输入后能输出完整结果表格。
- 平均价格计算器支持新增/删除买入行。
- 结果表格可复制。

## 阶段 5：接入 KLineCharts

目标：替换手写图表，完成正式 K 线展示基础。

开发内容：

- 安装并封装 KLineCharts。
- 渲染日 K：open、high、low、close、volume。
- 添加 MA、BOLL、VOL、KDJ、MACD 指标。
- 支持指标显示/隐藏。
- 支持 MA 参数配置。
- MA/BOLL 叠加在主 K 线图，VOL/KDJ/MACD 使用独立副图。
- 添加十字光标信息和当前复盘日竖线。
- 确保多指标同时开启时不会覆盖或挤压主 K 线。

验收标准：

- K 线能缩放和平移。
- MA、BOLL、KDJ、MACD、成交量能显示。
- 修改 MA 参数后图表更新。
- 鼠标移动能看到当前 K 线和指标值。
- 指标窗格与主图布局清晰，不互相覆盖。

## 阶段 6：实现 PostgreSQL 后端数据库

目标：建立正式长期存储能力。

开发内容：

- 使用 PostgreSQL 作为主数据库。
- 在 `apps/api` 中接入 SQLModel 或 SQLAlchemy 2.x。
- 配置数据库连接：
  - `.env`
  - `DATABASE_URL=postgresql+psycopg://stock_sim:stock_sim@localhost:5432/stock_sim`
- 接入 Alembic 迁移。
- 建立 PostgreSQL 初始化说明或 Docker Compose。
- 建立模型：
  - `Instrument`
  - `KlineDaily`
  - `WatchlistItem`
  - `ReplaySession`
  - `Trade`
  - `TradeReview`
  - `FeeTemplate`
- 建立基础 CRUD。
- 实现 `GET /api/health/db` 检查数据库连接。

验收标准：

- 本地 PostgreSQL 可连接。
- 运行迁移后能创建全部核心表。
- 后端启动时不会隐式丢表或重建表。
- 能通过 API 新增/查询标的、自选和复盘 session。
- 重启后端后数据仍存在。

## 阶段 6.1：数据库迁移与约束完善

目标：保证 PostgreSQL schema 可演进、可追踪、可防重复。

开发内容：

- 初始化 Alembic：
  - `alembic.ini`
  - `migrations/env.py`
  - 第一版 migration
- 为核心表添加外键、唯一约束和索引。
- 为 `kline_daily` 添加唯一约束：

```text
(instrument_id, trade_date, adjust_type, source)
```

- 为 `indicator_config`、`tags`、`metrics_snapshot`、`config` 使用 `JSONB`。
- 为金额、价格、数量使用 `numeric`。
- 为时间字段使用 `timestamptz`。

验收标准：

- `alembic upgrade head` 能成功创建 schema。
- `alembic downgrade -1` 可回滚最近一次迁移。
- 重复插入同一条 K 线会被唯一约束阻止或 upsert 更新。
- 表结构与 `docs/设计文档.md` 保持一致。

## 阶段 7：接入真实行情数据

目标：搜索股票/ETF 并同步历史日 K 到 PostgreSQL。

开发内容：

- 实现行情 provider 接口：

```text
search(keyword)
fetch_daily_bars(symbol, start_date, end_date, adjust_type)
```

- 第一版接 AKShare。
- 股票和 ETF 分开适配。
- 同步数据写入 `kline_daily`。
- 使用 PostgreSQL upsert 保证同步幂等。
- 返回最近同步时间、数据源、复权类型。
- 外部接口失败时有错误提示和重试。

验收标准：

- 搜索 `600519`、`000001`、`510300` 能返回结果。
- 能同步某个标的今天以前的历史日 K。
- 同步同一标的多次不会产生重复 K 线。
- 前端能从后端加载真实 K 线。
- 页面显示数据源、复权类型、最后同步时间。

## 阶段 8：实现复盘时间规则

目标：保证用户不会看到未来。

开发内容：

- 创建复盘 session 时选择起始日期。
- 后端或前端根据 `current_date` 截断 K 线。
- 实现前一日、后一日、跳转日期。
- 隐藏未来开关默认打开。
- 指标计算只使用当前复盘日及以前的数据。

验收标准：

- 当前复盘日之后的 K 线不可见。
- 每推进一天，图表和指标只多显示一天。
- 跳转到过去日期时，未来买卖点不可见或灰化。

## 阶段 9：实现交易引擎

目标：把“买高卖低”的训练规则正式固化。

开发内容：

- 交易表单支持：
  - 买入/卖出
  - 数量
  - 手续费
  - 笔记
  - 情绪评分，可选
- 买入价格自动取当日 `high`。
- 卖出价格自动取当日 `low`。
- 禁止卖出超过当前持仓。
- 使用 FIFO 计算持仓、平均成本和已实现盈亏。
- 交易记录写入 PostgreSQL。

验收标准：

- 买入记录价格等于当日最高价。
- 卖出记录价格等于当日最低价。
- 卖出超过持仓时有明确提示。
- 交易记录刷新后不丢失。
- PostgreSQL 中能查到对应交易。

## 阶段 10：实现持仓区域、买卖标记和笔记弹窗

目标：把交易过程清晰画在 K 线上。

开发内容：

- 使用 KLineCharts overlay 绘制 B/S 买卖点。
- 买入点显示 `B` 和“买入”。
- 卖出点显示 `S` 和“卖出”。
- 买入到卖出之间显示持仓高亮区域。
- 区域上显示本段盈亏百分比，例如 `+18%`。
- 点击买卖点弹出交易笔记。
- 未完成持仓从买入日高亮到当前复盘日。

验收标准：

- 买卖点在 K 线图上位置正确。
- 持仓区域高亮不会遮挡 K 线主体。
- 点击买卖点能看到笔记。
- 隐藏未来时，未来买卖点和未来区域不可见。

## 阶段 11：实现浮亏压力与痛感面板

目标：让买入后的亏损过程被持续看见。

开发内容：

- 显示：
  - 当前持仓
  - 平均成本
  - 当前收盘浮盈亏
  - 当前低点浮盈亏
  - 买入后最大浮亏
  - 已实现盈亏
  - 总盈亏
- 标出买入后最低低点。
- 增加浮亏压力条。
- 增加持仓期间每日低点盈亏曲线。
- 关键计算结果可缓存为 PostgreSQL 快照，便于统计复盘。

验收标准：

- 推进复盘日期时，浮盈亏逐日变化。
- 后续出现更低价时，最大浮亏更新。
- 即使尚未卖出，也能看到当前最差浮动结果。

## 阶段 12：实现区间复盘和笔记

目标：让用户总结一整段交易，而不是只记录单点。

开发内容：

- 支持选择起始买入记录和结束卖出记录。
- 计算区间投入、收入、收益、收益率、最大浮亏。
- 支持为区间写总结笔记。
- 支持给笔记添加标签。
- 区间复盘写入 `trade_reviews`。

验收标准：

- 能选择一段买入到卖出的交易过程。
- 系统正确计算该区间盈亏。
- 区间笔记刷新后不丢失。
- 标签能用于后续统计。

## 阶段 13：统计和训练反馈

目标：从“记录交易”升级到“发现自己的问题”。

开发内容：

- 统计总交易次数、胜率、平均盈利、平均亏损、盈亏比。
- 按标签统计亏损金额和次数。
- 统计常见错误：
  - 追高
  - 恐慌卖出
  - 未按计划
  - 仓位过重
  - 指标冲突仍交易
- 增加复盘日历。
- 复杂统计可使用 PostgreSQL 聚合查询或物化视图。

验收标准：

- 统计页能看到历史训练结果。
- 点击某个错误标签能回到对应交易记录。
- 能看出亏损最多的问题类型。

## 阶段 14：设置页、数据质量和费率模板

目标：让复盘和计算器结果可信。

开发内容：

- 设置页支持复权类型选择：
  - 不复权
  - 前复权
  - 后复权
- 显示数据缺口和停牌日期。
- 增加行情重新同步。
- 增加数据源配置：
  - AKShare
  - Tushare Pro
- 增加手续费规则配置：
  - 固定手续费
  - 按比例手续费
  - 最低手续费
  - 卖出印花税
  - 过户费
- 费率模板写入 PostgreSQL `fee_templates`。

验收标准：

- 用户知道当前使用什么数据源和复权方式。
- 重新同步不会产生重复 K 线。
- 费率配置变化后，复盘盈亏和计算器结果能重新计算。
- 费率模板刷新后不丢失。

## 阶段 16：打包与本地发布

目标：让自己可以稳定使用。

开发内容：

- 提供 Docker Compose：
  - PostgreSQL
  - FastAPI
  - Web
- 前端构建后可由后端托管静态文件，或由独立 Web 服务托管。
- PostgreSQL 数据目录使用 Docker volume。
- 增加备份和导出：
  - `pg_dump` 备份 PostgreSQL
  - 导出交易记录 CSV
  - 导出复盘笔记 Markdown
  - 导出计算器结果，可选
- 提供一键启动脚本和备份脚本。

验收标准：

- 本机可通过一个命令启动完整系统。
- 重启电脑后 PostgreSQL 数据仍在。
- 交易和笔记可以导出备份。
- 能从 `pg_dump` 备份恢复数据库。

## 推荐第一轮最小闭环

如果想最快开始可用，第一轮建议只做下面 10 件事：

1. 迁移到 React + TypeScript + Vite。
2. 建立左侧图标菜单和页面切换。
3. 做出计算器页，并完成利润成本、涨跌幅、平均价格计算器。
4. 用 KLineCharts 显示 mock K 线和指标。
5. 建立 FastAPI + PostgreSQL + Alembic。
6. 接 AKShare 拉取真实日 K 并写入 PostgreSQL。
7. 搜索并加入自选。
8. 创建复盘 session 并隐藏未来。
9. 实现买高卖低交易记录和 B/S 标记。
10. 实现持仓、盈亏、最大浮亏和笔记。

完成这 10 件事后，项目就从 demo 变成真正可用的个人复盘训练系统，并且具备长期保存数据的 PostgreSQL 基础。
