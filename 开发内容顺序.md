# 开发内容顺序

本文档按“能持续交付、每一步都有可验证结果”的方式组织。建议每完成一个阶段就运行测试、手动体验一次完整复盘，并提交代码。

## 阶段 0：整理当前 demo

目标：保护当前原型，明确它的价值和限制。

开发内容：

- 确认当前 `index.html`、`app.js`、`styles.css`、`serve.mjs` 能正常运行。
- 统一文件编码为 UTF-8，修复中文乱码。
- 给当前 demo 打一个说明：它是交互原型，不是正式架构。
- 截图或录屏记录当前核心体验，方便后续迁移时对照。

验收标准：

- 本地运行 `node serve.mjs` 能打开页面。
- 页面中文正常显示。
- 能完成一次假数据买入、卖出、笔记和盈亏查看。

## 阶段 1：建立正式项目骨架

目标：从静态 demo 进入可维护工程结构。

推荐结构：

```text
apps/
  web/
  api/
docs/
```

开发内容：

- 新建 `apps/web`：Vite + React + TypeScript。
- 新建 `apps/api`：FastAPI。
- 新建基础 README，写清前后端启动方式。
- 配置统一代码风格：ESLint、Prettier、ruff 或 black。
- 保留旧 demo 到 `legacy-demo/` 或 `prototype/`，避免迁移时丢失体验参考。

验收标准：

- 前端能启动并显示空工作台。
- 后端能启动并返回健康检查：`GET /api/health`。
- README 可以指导自己从零启动项目。

## 阶段 2：前端工作台布局迁移

目标：先把界面形状搭起来，不急着接真实数据。

开发内容：

- 拆分页面组件：
  - `Sidebar`
  - `InstrumentSearch`
  - `Watchlist`
  - `IndicatorSettings`
  - `ChartWorkspace`
  - `TradePanel`
  - `PnlPanel`
  - `TradeHistory`
- 使用 Zustand 保存当前标的、复盘日期、指标配置、图表开关。
- 暂时使用静态 mock 数据替代真实接口。

验收标准：

- 页面布局接近当前 demo。
- 所有控件能修改前端状态。
- 自选、指标开关、复盘日期在刷新前可正常交互。

## 阶段 3：接入 KLineCharts

目标：替换手写 Canvas，完成正式 K 线展示基础。

开发内容：

- 安装并封装 KLineCharts。
- 渲染日 K：open、high、low、close、volume。
- 添加 MA、BOLL、VOL、KDJ、MACD 指标。
- 支持指标显示/隐藏。
- 支持 MA 参数配置。
- 添加十字光标信息面板。
- 添加当前复盘日竖线。

验收标准：

- K 线能缩放、平移。
- MA、BOLL、KDJ、MACD、成交量能显示。
- 修改 MA 参数后图表更新。
- 鼠标移动能看到当前 K 线和指标值。

## 阶段 4：实现后端数据库

目标：让系统有长期存储能力。

开发内容：

- 配置 SQLite。
- 建立模型：
  - `Instrument`
  - `KlineDaily`
  - `WatchlistItem`
  - `ReplaySession`
  - `Trade`
  - `TradeReview`
- 建立数据库初始化脚本。
- 实现基础 CRUD。

验收标准：

- 后端启动时能创建数据库。
- 能通过 API 新增/查询标的、自选和复盘 session。
- 重启服务后数据仍然存在。

## 阶段 5：接入真实行情数据

目标：搜索股票/ETF 并同步历史 K 线。

开发内容：

- 实现行情数据 provider 接口：

```text
search(keyword)
fetch_daily_bars(symbol, start_date, end_date, adjust_type)
```

- 第一版接 AKShare。
- 股票和 ETF 分开适配。
- 同步数据写入 `KlineDaily`。
- 返回最近同步时间和数据源。
- 对外部接口失败做错误提示和重试。

验收标准：

- 搜索 `600519`、`000001`、`510300` 等标的能返回结果。
- 能同步某个标的今天以前的历史日 K。
- 前端能从后端加载真实 K 线。
- 页面显示复权类型、数据源、最后同步时间。

## 阶段 6：实现复盘时间规则

目标：保证用户不会看到未来。

开发内容：

- 创建复盘 session 时选择开始日期。
- 后端或前端根据 `current_date` 截断 K 线。
- 实现前一日、后一日、跳转日期。
- 隐藏未来开关默认打开。
- 指标计算只使用当前复盘日及以前的数据。

验收标准：

- 当前复盘日之后的 K 线不可见。
- 每推进一天，图表和指标只多显示一天。
- 跳转到过去日期时，未来买卖点也不显示或灰化处理。

## 阶段 7：实现交易引擎

目标：把“买高卖低”的训练规则正式固化。

开发内容：

- 交易表单支持：
  - 买入/卖出
  - 数量
  - 手续费
  - 笔记
  - 情绪评分，可选
- 买入价格自动取当日 high。
- 卖出价格自动取当日 low。
- 禁止卖出超过当前持仓。
- 使用 FIFO 计算持仓、平均成本和已实现盈亏。
- 在图表上显示买卖标记。

验收标准：

- 买入记录价格等于当日最高价。
- 卖出记录价格等于当日最低价。
- 卖出超过持仓时有明确提示。
- 图表买卖点清晰可见。
- 交易记录刷新后不丢失。

## 阶段 8：实现浮亏压力与痛感面板

目标：让买入后的亏损过程被持续看见。

开发内容：

- 显示：
  - 当前持仓
  - 平均成本
  - 当前收盘浮盈亏
  - 当前低点浮盈亏
  - 买入后最大浮亏
  - 已实现盈亏
  - 总盈亏
- 在图表或面板中标出买入后最低低点。
- 增加浮亏压力条。
- 增加“持仓期间每日低点盈亏曲线”。

验收标准：

- 买入后推进复盘日期，浮亏会逐日变化。
- 当后续出现更低价时，最大浮亏更新。
- 即使尚未卖出，也能直观看到当前最差浮动结果。

## 阶段 9：实现区间复盘和笔记

目标：让用户能总结一整段交易，而不是只记录单点。

开发内容：

- 支持选择起始买入记录和结束卖出记录。
- 计算区间投入、收入、收益、收益率、最大浮亏。
- 支持为区间写总结笔记。
- 支持给笔记添加标签。
- 交易历史中能展开查看区间复盘。

验收标准：

- 能选择一段买入到卖出的交易过程。
- 系统正确计算该区间盈亏。
- 区间笔记保存后刷新不丢失。

## 阶段 10：统计和训练反馈

目标：从“记录交易”升级到“发现自己的问题”。

开发内容：

- 统计总交易次数、胜率、平均盈利、平均亏损、盈亏比。
- 按标签统计亏损金额和次数。
- 统计最常见错误：
  - 追高
  - 恐慌卖出
  - 未按计划
  - 仓位过重
  - 指标冲突仍交易
- 增加复盘日历。

验收标准：

- 有一个统计页面能看到历史训练结果。
- 点击某个错误标签能回到对应交易记录。
- 能看出自己亏损最多的问题类型。

## 阶段 11：数据质量和配置

目标：让复盘结果可信。

开发内容：

- 支持复权类型选择：不复权、前复权、后复权。
- 显示数据缺口和停牌日期。
- 增加行情重新同步。
- 增加数据源配置：
  - AKShare
  - Tushare Pro
- 增加手续费规则配置：
  - 固定手续费
  - 按比例手续费
  - 最低手续费
  - 卖出印花税

验收标准：

- 用户知道当前使用什么数据源和复权方式。
- 重新同步不会产生重复 K 线。
- 手续费配置变化后，盈亏能重新计算。

## 阶段 12：测试与稳定性

目标：避免核心计算出错。

开发内容：

- 后端单元测试：
  - MA/BOLL/KDJ/MACD
  - 买入最高价
  - 卖出最低价
  - FIFO 盈亏
  - 最大浮亏
- 前端单元测试：
  - 状态切换
  - 表单校验
  - 指标配置
- Playwright 端到端测试：
  - 搜索标的
  - 加入自选
  - 同步 K 线
  - 创建复盘
  - 买入卖出
  - 查看盈亏和笔记

验收标准：

- 核心计算都有测试覆盖。
- 一条完整复盘流程可以自动化跑通。
- 修复 bug 时优先补测试。

## 阶段 13：打包与本地发布

目标：让自己可以稳定使用。

开发内容：

- 提供一键启动脚本。
- 前端构建后由后端托管静态文件，或使用 Docker Compose 同时启动。
- 数据库文件放到固定目录。
- 增加备份和导出：
  - 导出交易记录 CSV。
  - 导出复盘笔记 Markdown。
  - 备份 SQLite 数据库。

验收标准：

- 在本机可以通过一个命令启动完整系统。
- 重启电脑后数据还在。
- 交易和笔记可以导出备份。

## 推荐第一轮最小闭环

如果想最快开始开发，第一轮只做下面 8 件事：

1. 迁移到 React + TypeScript + Vite。
2. 用 KLineCharts 显示 mock K 线和指标。
3. 建 FastAPI + SQLite。
4. 接 AKShare 拉取真实日 K。
5. 搜索并加入自选。
6. 创建复盘 session 并隐藏未来。
7. 实现买高卖低交易记录。
8. 实现持仓、盈亏、最大浮亏和笔记。

完成这 8 件事后，这个项目就从 demo 变成了真正可用的个人训练系统。
